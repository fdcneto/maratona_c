A Trindade Básica

Input, Output e Retorno.

1. 'return 0;'

É o "aviso" do seu programa (o 'main') para o Sistema Operacional. Significa: "Chefe, terminei e deu tudo certo!". Um 'return 1;' significaria erro.

2. 'scanf()'

O oposto do 'printf'. Se 'printf' *escreve* na tela, 'scanf' *lê* do teclado. Ele pega o que o usuário digita e guarda em uma variável.

// O programa agora é interativo!
#include <stdio.h>
#include <stdlib.h>

int main(void)
{
    int idade; // 1. Cria uma "caixa" (variável)

    printf("Digite sua idade: ");
    
    // 2. Lê um inteiro (%d) e guarda
    //    no ENDEREÇO (&) de 'idade'
    scanf("%d", &idade);
    
    printf("Voce tem %d anos!\n", idade);
    
    system("pause");
    return 0; // 3. "Tudo certo!"
}

As "Etiquetas" Mágicas

Especificadores de Formato ('%')

O 'scanf' e o 'printf' não são "espertos". Você **precisa** dizer a eles que tipo de dado está lendo ou escrevendo. Essas "etiquetas" ('%') são **obrigatórias**.

Etiqueta Tipo de Dado O que representa
%d (ou %i) int Número **Inteiro**
%f float Número **Decimal** (ex: 1.75)
%c char Um **ÚNICO Caractere** (ex: 'A')
%s char[] Um **Texto (String)** (ex: "Maria")
⚠️ A MAIOR PEGADINHA DE C!
Para 'printf': Use %f para 'float' e 'double'.
Para 'scanf': Use %f para 'float', mas use %lf (long float) para 'double'.

Exemplo Prático 1

Juntando as Etiquetas.

Este programa usa todos os 4 especificadores essenciais para criar uma "Ficha Cadastral" básica.

  • %s para o nome (um array 'char[]').
  • %d para a idade ('int').
  • %f para a altura ('float').
  • %c para a inicial ('char').
  • Note o espaço antes do '%c' no 'scanf'! É um truque para ele ignorar o "Enter" da digitação anterior.
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    char nome[50];
    int idade;
    float altura;
    char inicial;

    printf("Nome: ");
    scanf("%s", nome); // NÃO usa &
    
    printf("Idade: ");
    scanf("%d", &idade); // Usa &
    
    printf("Altura (ex: 1.75): ");
    scanf("%f", &altura); // Usa &
    
    printf("Inicial: ");
    scanf(" %c", &inicial); // Usa & e o espaço!

    printf("\n--- Ficha ---\n");
    printf("Nome: %s (%c)\n", nome, inicial);
    printf("Idade: %d\n", idade);
    printf("Altura: %.2f\n", altura); // %.2f = 2 casas

    system("pause");
    return 0;
}

Controle de Fluxo

Repetindo Tarefas: 'while' e 'for'

Loops servem para você não ter que escrever o mesmo código várias vezes.

1. 'while' (Enquanto)

O loop mais simples. "Enquanto a condição for VERDADEIRA, continue executando". Você precisa controlar o "passo" (ex: 'i++') manually.

2. 'for' (Para)

O especialista em contagem. Ele junta as 3 partes essenciais de um loop em uma só linha:

for ( 1. Início ; 2. Condição ; 3. Passo )

// Contando de 1 a 5 com WHILE
int i = 1; // 1. Início
while (i <= 5) { // 2. Condição
    printf("%d\n", i);
    i++; // 3. Passo (ESSENCIAL!)
}

// Contando de 1 a 5 com FOR
// ( 1. Início ; 2. Condição ; 3. Passo )
for (int j = 1; j <= 5; j++) {
    printf("%d\n", j);
}

Guardando Dados (1D)

'Arrays' (Listas)

E se você precisar guardar 5 notas? 'nota1', 'nota2'... é péssimo.

Use um Array. É uma única variável que funciona como um "gaveteiro" com vários espaços para dados do mesmo tipo.

O Conceito Base-Zero!

A contagem em C (e na maioria das linguagens) começa em ZERO.

Um array de 5 posições 'int notas[5]' tem os índices:

notas[0], notas[1], notas[2], notas[3], notas[4]

Tentar acessar 'notas[5]' é um erro grave!

#include <stdio.h>
#include <stdlib.h>

int main(void) {
    // Declara um "gaveteiro" para 3 notas
    float notas[3]; 
    float soma = 0;

    // O 'for' é perfeito para percorrer arrays!
    for (int i = 0; i < 3; i++) {
        printf("Digite a nota %d: ", i + 1);
        // Lê e guarda em notas[0], notas[1]...
        scanf("%f", &notas[i]); 
        soma += notas[i]; // soma = soma + notas[i]
    }
    
    printf("A media e: %.1f\n", soma / 3);

    system("pause");
    return 0;
}

Guardando Dados (2D)

'Matrizes' (Tabelas)

Se um Array é um gaveteiro (1D), uma Matriz é uma "estante de gaveteiros" (2D). É um "array de arrays".

É perfeito para representar tabelas, planilhas ou tabuleiros. Você precisa de 2 índices: '[LINHA][COLUNA]'.

Loops Aninhados

Para percorrer uma Matriz (2D), você precisa de 2 loops (um dentro do outro):

  • O loop "de fora" (pai) controla as Linhas.
  • O loop "de dentro" (filho) controla as Colunas.
// Matriz para 2 alunos, 3 notas cada
float notas[2][3]; // [Linhas][Colunas]

// Loop "pai" para as LINHAS (alunos)
for (int aluno = 0; aluno < 2; aluno++) {
    
    printf("--- Aluno %d ---\n", aluno + 1);
    
    // Loop "filho" para as COLUNAS (notas)
    for (int nota = 0; nota < 3; nota++) {
        
        printf("Digite a nota %d: ", nota + 1);
        scanf("%f", &notas[aluno][nota]);
    }
}

Organizando o Caos

'Funções' (O "Gerente" e os "Especialistas")

Sua função 'main' vai ficar gigante e confusa. Funções são "mini-programas" que você cria para organizar o código.

A 'main' vira o "Gerente" e as funções são os "Especialistas" que o gerente chama.

  • **Reutilização:** Escreva o código 'somar()' uma vez, use 100 vezes.
  • **Organização:** Seu 'main' fica limpo e legível.

Sintaxe

tipo_de_retorno nome_da_funcao ( tipo parametro1, ... ) {
  // Código...
  return valor_de_retorno;
} 
#include <stdio.h>
#include <stdlib.h>

// --- 1. O "Especialista" ---
// Retorna um 'int', recebe dois 'int'
int somar(int a, int b) {
    int resultado = a + b;
    return resultado; // Devolve a resposta
}

// --- 2. O "Gerente" ---
int main(void) {
    int n1 = 10;
    int n2 = 20;
    
    // 3. O Gerente "chama" o especialista
    int resposta = somar(n1, n2);
    
    printf("A soma e: %d\n", resposta); // 30
    
    system("pause");
    return 0;
}

Conceito Avançado

'Recursividade'

É uma função que **chama a si mesma** para resolver um problema, quebrando-o em uma versão menor *exatamente do mesmo problema*.

Pense em Bonecas Russas (Matryoshka). Seu problema é "abrir todas as bonecas". Você abre uma, encontra uma boneca menor, e aplica a *mesma regra* ("abrir boneca") para a menor.

As 2 Regras de Ouro

  1. O Caso Base: A condição de parada. A "boneca" final que não abre mais. (Ex: 'n == 1').
  2. O Passo Recursivo: A chamada para si mesma, com um problema menor. (Ex: 'n * fatorial(n-1)').

Se faltar o Caso Base, você cria um "loop infinito" (Stack Overflow)!

// Exemplo: Fatorial (5! = 5*4*3*2*1)
int fatorial(int n) {
    // 1. O CASO BASE (A boneca final)
    if (n == 1) {
        return 1;
    }
    // 2. O PASSO RECURSIVO
    else {
        return n * fatorial(n - 1);
    }
}

Criando Seus Próprios Tipos

'Structs' (Estruturas)

E se você quiser agrupar dados de tipos **DIFERENTES**? (Ex: nome, idade e média de um aluno).

Um 'array' não serve (só aceita 1 tipo). A solução é uma 'struct'.

Pense nela como uma "Ficha Cadastral" ou um "Molde". Você define o molde 'struct Aluno' uma vez, e depois pode criar quantas fichas (variáveis) quiser.

#include <stdio.h>
#include <string.h> // Para strcpy

// --- 1. Definindo o "MOLDE" ---
// (Geralmente fora da main)
struct Aluno {
    char nome[50];
    int idade;
    float media;
}; // <-- Ponto e vírgula aqui!

int main(void) {
    // --- 2. Criando uma "FICHA" ---
    struct Aluno aluno1;

    // --- 3. Acessando os campos ---
    // Use o PONTO (.)
    aluno1.idade = 20;
    aluno1.media = 8.5;
    strcpy(aluno1.nome, "Joao Silva");

    printf("Nome: %s\n", aluno1.nome);
    printf("Idade: %d\n", aluno1.idade);
    
    return 0;
}

Juntando os Conceitos

Arrays + Structs

E se você quiser cadastrar uma **turma inteira** de 30 alunos?

Você cria um Array onde cada item é uma Struct.

// 1. Definir o Molde
struct Aluno {
    char nome[50];
    float media;
};

int main(void) {
    // 2. Criar um ARRAY de 30 Structs
    struct Aluno turma[30];
    
    // 3. Acessar os dados
    
    // Acessa o PRIMEIRO aluno (índice 0)
    strcpy(turma[0].nome, "Maria");
    turma[0].media = 9.0;
    
    // Acessa o TERCEIRO aluno (índice 2)
    strcpy(turma[2].nome, "Carlos");
    turma[2].media = 7.5;
    
    printf("Nome do Aluno 3: %s\n", turma[2].nome);
    
    return 0;
}

O Coração do C

'Ponteiros' (O Conceito)

Este é o tópico mais importante de C. A ideia é simples:

Um ponteiro é uma variável que não guarda um valor, mas sim o ENDEREÇO DE MEMÓRIA de outra variável.

Analogia: Casa vs. Endereço

Quando você cria 'int idade = 30;':

  • Você construiu uma Casa ('idade').
  • O conteúdo da casa é '30'.
  • A casa tem um Endereço (ex: 'nº 1234' da Rua Memória).

Um **Ponteiro** ('int* pIdade;') é um papel em branco feito para anotar o *endereço* ('nº 1234') dessa casa.

CASA
int idade = 30;
Endereço: '1234'
ENDEREÇO
int* pIdade;
Conteúdo: '1234'

Os 2 Operadores de Ponteiro

É aqui que 99% das pessoas se confundem.

&

Operador "Endereço de..."

É a ferramenta que **lê o endereço** de uma variável (a "casa").

int idade = 30;
int* pIdade; // Papel em branco

// O "papel" recebe o "ENDEREÇO DE" idade
pIdade = &idade; 

// Agora pIdade vale "1234"
// (Você já usou isso: scanf("%d", &idade))

*

Operador "Valor em..." (Dereferência)

É a ferramenta que **"vai até o endereço"** e pega o valor que está lá.

// pIdade tem o endereço "1234"

// Imprime o valor DO ponteiro (o endereço)
printf("%p", pIdade); // Imprime: 1234

// Imprime o VALOR NO ENDEREÇO
printf("%d", *pIdade); // Imprime: 30

// Modifica o valor NO ENDEREÇO
*pIdade = 40;
printf("%d", idade); // Agora 'idade' vale 40!

Por que Usar Ponteiros?

Para Modificar Variáveis em Funções.

Por padrão, C passa variáveis para funções por CÓPIA. A função modifica a cópia, não o original.

Para modificar o original, você não passa a "casa", você passa o "endereço" (o ponteiro).

Isso é chamado de **"Passagem por Referência"**.

// 1. A função NÃO funciona (passa cópia)
void nao_muda(int x) {
    x = 100; // Modifica a CÓPIA
}

// 2. A função FUNCIONA (passa endereço)
// Ela aceita um "papel de endereço"
void muda_de_verdade(int* pX) {
    // "Vá até o endereço e guarde 100 lá"
    *pX = 100;
}

int main(void) {
    int valor = 10;
    
    nao_muda(valor);
    printf("%d\n", valor); // Imprime 10
    
    muda_de_verdade(&valor); // Passa o ENDEREÇO
    printf("%d\n", valor); // Imprime 100
    
    return 0;
}

Por que Usar Ponteiros?

Para Eficiência (Evitar Cópia)

Imagine que sua 'struct Aluno' tem 50 campos. Se você passar ela para uma função...

LENTO E RUIM: Passar por Cópia

'void func(struct Aluno aluno)'

O C COPIA TODOS OS 50 CAMPOS. É lento e gasta memória. A função não pode modificar o original.

RÁPIDO E BOM: Passar por Ponteiro

'void func(struct Aluno* pAluno)'

O C COPIA SÓ O ENDEREÇO (um "papelzinho"). É super rápido. A função pode modificar o original.

A Grande Revelação

Arrays JÁ SÃO Ponteiros!

Em C, o nome de um array não é o array... é um ponteiro constante para o primeiro elemento.

int notas[10];

// 'notas' é SÓ um ponteiro
// que aponta para notas[0].

// É por isso que você FAZ:
scanf("%d", &notas[0]); // Certo

// Mas também pode fazer:
scanf("%d", notas); // Certo (notas == ¬as[0])

// E é por isso que em strings:
char nome[50];
scanf("%s", nome); // Certo!
scanf("%s", &nome); // ERRADO! (É um & de um &)

A Ferramenta Final

O Operador Seta '->'

Ok, temos um ponteiro para uma struct ('struct Aluno* pAluno'). Como acessamos a 'media'?

Com a Struct (Normal)

Se você tem a "casa", use o PONTO ('.').

struct Aluno aluno1;
aluno1.media = 9.0;

Com o Ponteiro (Novo)

Se você tem o "endereço", use a SETA ('->').

struct Aluno* pAluno = &aluno1;
pAluno->media = 9.0;

A '->' é um atalho que significa: "Vá até o endereço e pegue o campo da struct".
pAluno->media é o mesmo que (*pAluno).media

O Grande Final

Juntando TUDO.

Este programa prova todos os conceitos:

  1. Define uma 'struct'.
  2. Cria uma função 'darPontoExtra' que aceita um 'ponteiro' para essa struct (para ser eficiente e poder modificar o original).
  3. Usa o operador Seta ('->') para acessar o campo da struct original.
  4. Chama a função passando o Endereço ('&') da variável.

O resultado é que a 'main' vê o valor alterado pela função.

#include <stdio.h>
#include <stdlib.h>

// 1. O Molde
struct Aluno {
    float media;
};

// 2. A Função (aceita um endereço)
void darPontoExtra(struct Aluno* pAluno) {
    printf("[Funcao] Adicionando 1.0...\n");
    
    // 3. Usa a SETA (->) para modificar
    pAluno->media += 1.0; 
}

int main(void) {
    struct Aluno aluno1;
    aluno1.media = 7.5;
    
    printf("[Main] Media antes: %.1f\n", aluno1.media);
    
    // 4. Passa o ENDEREÇO (&)
    darPontoExtra(&aluno1);
    
    printf("[Main] Media DEPOIS: %.1f\n", aluno1.media);
    
    system("pause");
    return 0;
}

Tópico Avançado 1

Alocação Dinâmica ('malloc')

**O Problema:** 'int notas[10];' tem tamanho fixo. E se você não sabe o tamanho? E se o *usuário* decide que quer 50 notas?

**A Solução:** "Alugar" memória em tempo de execução. 'malloc' significa "Memory Allocation".

  • 'malloc(bytes)': Pede ao SO um bloco de memória do tamanho em bytes.
  • 'sizeof(tipo)': Usado para calcular os bytes. (Ex: '10 * sizeof(int)').
  • 'free(ponteiro)': **OBRIGATÓRIO!** Devolve a memória alugada. Esquecer isso causa "Memory Leak" (vazamento de memória).
#include <stdio.h>
#include <stdlib.h> // Para malloc e free

int main(void) {
    int tamanho;
    printf("Quantas notas quer digitar? ");
    scanf("%d", &tamanho);
    
    // 1. Alugar a memória
    // (int*) -> "Converte" o ponteiro genérico
    int* notas = (int*) malloc(tamanho * sizeof(int));
    
    // 2. Checar se a alocação deu certo
    if (notas == NULL) {
        printf("Erro! Sem memoria!\n");
        return 1;
    }
    
    // 3. Usar a memória (como um array normal)
    for(int i = 0; i < tamanho; i++) {
        notas[i] = i * 10;
        printf("Nota: %d\n", notas[i]);
    }
    
    // 4. DEVOLVER a memória!
    free(notas);
    
    system("pause");
    return 0;
}

Tópico Avançado 2

Manipulação de Arquivos ('fopen')

Quando seu programa fecha, todas as variáveis ('int', 'struct') morrem. Para salvar dados permanentemente, você usa arquivos.

' FILE*' é um ponteiro especial que age como a "chave" para um arquivo no seu HD.

  • 'fopen("nome.txt", "w")': Abre o arquivo.
    • '"w"' (write): Apaga o conteúdo e escreve do zero.
    • '"r"' (read): Lê o arquivo.
    • '"a"' (append): Escreve no *fim* do arquivo.
  • 'fprintf(arq, ...)': Igual ao 'printf', mas para o arquivo.
  • 'fclose(arq)': **OBRIGATÓRIO!** "Fecha a chave" e salva as mudanças.
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    // 1. Tenta abrir/criar o arquivo
    FILE* arquivo = fopen("log.txt", "a"); // "a" = append
    
    if (arquivo == NULL) {
        printf("Erro ao abrir arquivo!\n");
        return 1;
    }
    
    int pontuacao = 1500;
    
    // 2. Escreve no arquivo
    fprintf(arquivo, "Nova pontuacao: %d\n", pontuacao);
    
    printf("Log salvo em log.txt!\n");
    
    // 3. Fecha o arquivo
    fclose(arquivo);
    
    system("pause");
    return 0;
}

Tópico Avançado 3

Listas Encadeadas (O Fim)

Este é o chefão final da base de C. Combina TUDO: 'Structs', 'Ponteiros' e 'malloc'.

**O Problema:** Um Array 'int arr[10]' tem tamanho fixo. Um 'malloc' também é (você aloca 10 e pronto). E se você quiser adicionar o 11º, 12º... item de forma flexível?

**A Solução:** Uma "corrente" de structs. Cada "elo" ('struct Node') guarda um dado E um ponteiro para o *próximo elo*.

Analogia: Caça ao Tesouro

Cada 'struct Node' é uma pista que contém:

  1. Um pedaço do tesouro (o 'dado').
  2. O endereço da próxima pista (o ponteiro 'proximo').

A última pista aponta para 'NULL' (fim). Você usa 'malloc' para criar uma nova pista quando precisa.

// A "forma" da pista (elo)
struct Node {
    int dado; // O "tesouro"
    
    // O "endereço" da próxima pista
    // Um ponteiro para si mesmo!
    struct Node* proximo; 
};

int main(void) {
    // Ponteiro para o início da corrente
    struct Node* cabeca = NULL;
    
    // 1. Aloca o primeiro elo (pista)
    cabeca = (struct Node*) malloc(sizeof(struct Node));
    cabeca->dado = 10;
    cabeca->proximo = NULL; // É o último (por enquanto)
    
    // 2. Aloca o segundo elo
    struct Node* segundo = (struct Node*) malloc(sizeof(struct Node));
    segundo->dado = 20;
    segundo->proximo = NULL;
    
    // 3. "Engata" o primeiro no segundo
    cabeca->proximo = segundo;
    
    // Agora a lista é: cabeca -> segundo -> NULL
    
    return 0;
}
